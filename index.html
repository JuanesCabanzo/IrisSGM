Sistema de Gestión Médica con Autenticación Biométrica Simulada

Aplicación web educativa para el registro y gestión de historiales clínicos que incorpora una simulación de autenticación biométrica mediante imagen del iris o rostro. Diseñada para desarrollarse progresivamente durante el tecnólogo.

Ver detalles del proyecto

Descripción del proyecto

El Sistema de Gestión Médica con Autenticación Biométrica Simulada es una plataforma web que permite el registro, almacenamiento y consulta de historiales médicos. En lugar de realizar análisis biomédicos o diagnósticos automáticos, el sistema guarda imágenes del iris o rostro como identificador visual (autenticación simulada) y ofrece mecanismos seguros de acceso por roles (paciente, médico, administrador).

El objetivo es construir una herramienta formativa y escalable que permita demostrar, en la sustentación y las entregas parciales, capacidades en diseño de bases de datos, desarrollo web, manejo de roles, seguridad básica y generación de reportes. En fases posteriores se podrá integrar reconocimiento biométrico real o módulos experimentales de visión por computador como extensión del proyecto.

Objetivo general

Desarrollar una aplicación web para la gestión de información clínica con autenticación biométrica simulada, que permita la administración de pacientes, la consulta de historiales y la generación de reportes básicos, asegurando control de accesos por roles.

Objetivos específicos

  • Diseñar la estructura de base de datos para pacientes, usuarios y historiales.
  • Implementar autenticación con roles (paciente, médico, administrador).
  • Permitir el registro y almacenamiento de imágenes del iris o rostro como elemento de identificación visual.
  • Proveer interfaces para que médicos registren diagnósticos manuales y pacientes consulten su historial.
  • Generar reportes estadísticos simples sobre la actividad del sistema.

Requisitos funcionales (resumen)

  1. Registrar usuarios: pacientes, médicos y administradores.
  2. Autenticación tradicional (usuario + contraseña) y visualización de imagen biométrica simulada.
  3. Consulta y gestión de historiales clínicos.
  4. Registro manual de diagnósticos y citas médicas por parte del médico.
  5. Generación de reportes y estadísticas básicas.
  6. Control de accesos y permisos según rol.

Requisitos no funcionales

  • Interfaz intuitiva y responsive.
  • Cifrado básico de credenciales.
  • Persistencia de datos en base de datos (MySQL) en fases posteriores.
  • Capacidad de ejecutarse en entorno local o servidor institucional.

Tecnologías propuestas

Frontend: HTML5, CSS3, JavaScript.

Backend (etapas posteriores): Java (NetBeans, Spring Boot) o Node.js.

Base de datos: MySQL.

Herramientas complementarias: Chart.js para gráficos, librerías de cifrado y utilidades para manejo de imágenes. Para demostraciones locales se puede usar localStorage.

Notas para la sustentación

  • Explicar claramente que la autenticación biométrica es simulada (imagen guardada y verificación de igualdad) para evitar obligaciones éticas y legales.
  • Documentar cada entrega: decisiones técnicas, código, pruebas y mejoras pendientes.
  • Mostrar roadmap de evolución y posibles integraciones (reconocimiento real, cumplimiento de normativas de salud).
